# 水力模型团队 L1-L5 能力评级详细标准

> **适用范围：** 水务企业水力模型团队能力评估与人员发展
> **制定依据：** 参考 CIWEM UDG Framework、FEMA Guidelines、ARR 国际标准
> **版本：** v1.0

---

## 评级体系概述

### 评级维度

| 维度 | 权重 | 说明 |
|------|------|------|
| **技术知识 (Knowledge)** | 25% | 水力学、水文学、软件原理等理论基础 |
| **实践技能 (Skills)** | 30% | 模型构建、验证、分析等实际操作能力 |
| **项目经验 (Experience)** | 25% | 项目复杂度、角色、成果 |
| **质量意识 (Quality)** | 10% | 验证流程、文档规范、质量检查 |
| **协作沟通 (Communication)** | 10% | 团队协作、客户沟通、知识分享 |

### 等级定义

| 等级 | 称号 | 英文 | 核心特征 |
|------|------|------|---------|
| L1 | 入门学徒 | Apprentice | 学习阶段，需密切指导 |
| L2 | 初级工程师 | Junior Engineer | 掌握基础，有限独立 |
| L3 | 中级工程师 | Professional Engineer | 独立完成常规任务 |
| L4 | 高级工程师 | Senior Engineer | 指导他人，处理复杂项目 |
| L5 | 专家工程师 | Expert Engineer | 行业权威，战略决策 |

---

## L1 - 入门学徒 (Apprentice)

### 总体描述
处于学习阶段的初学者，对水力建模有基本认识，需在密切监督下工作。主要任务是学习基础理论和工具使用。

### 能力要求详情

#### 1. 技术知识 (Knowledge)

| 知识点 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 水力学基础 | ★★☆☆☆ | 理解连续性方程、能量方程基本概念 |
| 水文学基础 | ★★☆☆☆ | 了解降雨-径流关系、汇流概念 |
| 排水系统认知 | ★★☆☆☆ | 认识管网、泵站、调蓄设施等基本设施 |
| 软件界面熟悉 | ★★☆☆☆ | 熟悉主流软件界面和基本操作 |
| 数据格式 | ★☆☆☆☆ | 了解常见的数据格式（CAD、GIS、Excel） |

**学习路径：**
- 完成公司内部培训课程
- 学习软件官方教程
- 阅读 2-3 个历史项目报告

#### 2. 实践技能 (Skills)

| 技能项 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 数据整理 | ★★☆☆☆ | 在指导下完成数据清洗和格式转换 |
| 简单模型操作 | ★★☆☆☆ | 能打开、浏览、运行现有模型 |
| 结果查看 | ★★☆☆☆ | 能查看结果图表，识别明显异常 |
| 地图制作 | ★☆☆☆☆ | 在模板基础上制作简单结果图 |
| 文档编辑 | ★★☆☆☆ | 能按模板编写基础文档 |

**典型任务：**
- 整理管网数据表格
- 协助绘制模型示意图
- 运行标准案例并记录结果
- 校对技术报告

#### 3. 项目经验 (Experience)

| 指标 | 要求 |
|------|------|
| 参与项目数 | ≥1 个（学习性质） |
| 项目角色 | 辅助/观察员 |
| 项目类型 | 简单的小型项目或培训案例 |
| 工作年限 | 0-1 年 |

#### 4. 质量意识 (Quality)

- ☑️ 了解公司质量管理体系
- ☑️ 知晓模型验证的基本概念
- ☑️ 能识别明显的数据错误（如负数管径）
- ☑️ 养成备份工作文件的习惯

#### 5. 协作沟通 (Communication)

- ☑️ 能清晰描述工作中遇到的问题
- ☑️ 主动向上级汇报进度和困难
- ☑️ 参与团队技术分享会（旁听）
- ☑️ 记录学习笔记

### L1 晋级要求

完成以下全部条件可晋升 L2：
- [ ] 通过公司内部技术基础考试（≥80分）
- [ ] 独立完成 1 个简单模型操作练习
- [ ] 参与 2 个实际项目（至少 100 工时）
- [ ] 获得直属上级推荐

## L2 - 初级工程师 (Junior Engineer)

### 总体描述
掌握水力建模基础技能，能在有限监督下完成标准化任务。开始承担项目中具体的建模任务，但需要定期检查和指导。

### 能力要求详情

#### 1. 技术知识 (Knowledge)

| 知识点 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 1D 水力模型原理 | ★★★☆☆ | 理解圣维南方程、曼宁公式、节点-管段概念 |
| 降雨模型 | ★★★☆☆ | 了解设计暴雨、雨型、强度-历时-频率关系 |
| 径流模型 | ★★★☆☆ | 掌握径流系数法、时间-面积图等基础方法 |
| 软件建模逻辑 | ★★★☆☆ | 理解软件的数据结构和计算流程 |
| GIS 基础 | ★★☆☆☆ | 能进行基本的空间数据操作 |
| 行业标准 | ★★☆☆☆ | 了解国标、地方标准的适用范围 |

**学习路径：**
- 完成至少 1 个完整的小模型构建
- 学习 2 种以上建模软件
- 参加外部专业培训（≥40 学时）

#### 2. 实践技能 (Skills)

| 技能项 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 1D 模型构建 | ★★★☆☆ | 能独立完成简单管网模型的搭建 |
| 基础数据准备 | ★★★☆☆ | 独立完成地形、管网、降雨等数据准备 |
| 参数设置 | ★★★☆☆ | 能根据指南设置合理的模型参数 |
| 模型运行调试 | ★★★☆☆ | 能解决常见的运行错误 |
| 结果基础分析 | ★★☆☆☆ | 能提取关键指标，制作标准图表 |
| 报告初稿撰写 | ★★☆☆☆ | 能按模板完成报告初稿 |

**典型任务：**
- 独立完成小型管网模型（<500 节点）
- 准备模型输入数据（CAD/GIS 处理）
- 运行模型并解决简单报错
- 制作标准结果图和表格

#### 3. 项目经验 (Experience)

| 指标 | 要求 |
|------|------|
| 参与项目数 | ≥3 个 |
| 负责模型数 | ≥2 个（小型模型） |
| 项目角色 | 模型工程师 |
| 项目类型 | 排水规划、简单评估项目 |
| 工作年限 | 1-3 年 |

#### 4. 质量意识 (Quality)

- ☑️ 执行公司模型验证检查清单
- ☑️ 进行基础的质量检查（质量平衡、合理性检查）
- ☑️ 记录建模过程中的关键决策
- ☑️ 对发现的明显错误进行修正
- ☑️ 参与内部质量评审（作为汇报人）

#### 5. 协作沟通 (Communication)

- ☑️ 定期向项目负责人汇报进展
- ☑️ 能清晰描述模型设置和假设
- ☑️ 参与团队技术讨论并提出问题
- ☑️ 协助 L1 人员学习和成长
- ☑️ 完成技术文档的编写

### L2 晋级要求

完成以下全部条件可晋升 L3：
- [ ] 独立完成 3 个以上中型模型（500-2000 节点）
- [ ] 模型通过内部质量评审（无重大缺陷）
- [ ] 参与 5 个以上实际项目
- [ ] 获得项目负责人的正面评价
- [ ] 通过中级技术能力评估

---

## L3 - 中级工程师 (Professional Engineer)

### 总体描述
具备独立完成常规水力建模任务的能力，能处理中等复杂度的项目。开始参与模型验证和校准工作，能够指导初级工程师。

### 能力要求详情

#### 1. 技术知识 (Knowledge)

| 知识点 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 高级 1D 建模 | ★★★★☆ | 精通复杂管网、多系统耦合建模 |
| 2D 水力建模 | ★★★☆☆ | 了解 2D 模型原理，能进行基础应用 |
| 模型验证理论 | ★★★★☆ | 掌握验证方法论、误差分析、统计指标 |
| 水质模型 | ★★☆☆☆ | 了解水质模型基本概念和应用场景 |
| 不确定性分析 | ★★★☆☆ | 理解敏感性分析、参数不确定性 |
| 行业标准深度 | ★★★☆☆ | 熟悉并能应用相关标准进行设计 |
| 编程基础 | ★★☆☆☆ | 能编写简单的数据处理脚本 |

**学习路径：**
- 学习 2D 建模软件
- 深入研究模型验证理论
- 学习 Python/MATLAB 等工具
- 参加进阶培训（≥60 学时）

#### 2. 实践技能 (Skills)

| 技能项 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 复杂 1D 模型 | ★★★★☆ | 能构建大型复杂管网模型（>2000 节点） |
| 2D 模型应用 | ★★★☆☆ | 能进行基础的 2D 建模和结果分析 |
| 模型校准 | ★★★★☆ | 能根据监测数据进行模型校准 |
| 验证报告 | ★★★☆☆ | 编写完整的模型验证报告 |
| 问题诊断 | ★★★★☆ | 能诊断并解决复杂模型问题 |
| 数据处理 | ★★★☆☆ | 能处理和分析大量监测数据 |
| 技术方案 | ★★★☆☆ | 能制定技术方案和建模策略 |

**典型任务：**
- 负责中型项目的主模型（2000-5000 节点）
- 进行模型校准和验证
- 解决复杂的技术问题
- 指导 L1-L2 工程师
- 参与技术方案制定

#### 3. 项目经验 (Experience)

| 指标 | 要求 |
|------|------|
| 参与项目数 | ≥8 个 |
| 负责主模型数 | ≥5 个（中型以上） |
| 校准项目数 | ≥2 个（有实测数据） |
| 项目角色 | 主模型工程师 |
| 项目类型 | 综合规划、详细设计 |
| 工作年限 | 3-6 年 |

#### 4. 质量意识 (Quality)

- ☑️ 制定项目特定的质量检查流程
- ☑️ 进行全面的模型验证（水文、水力、水质）
- ☑️ 评估模型不确定性并提出改进建议
- ☑️ 审核 L1-L2 工程师的模型工作
- ☑️ 建立和维护项目技术文档
- ☑️ 参与公司质量标准制定

#### 5. 协作沟通 (Communication)

- ☑️ 与客户进行技术沟通
- ☑️ 向非技术人员解释复杂的技术概念
- ☑️ 主持项目技术讨论会
- ☑️ 培训 L1-L2 工程师
- ☑️ 撰写技术博客或内部分享
- ☑️ 参与跨部门协作

### L3 晋级要求

完成以下全部条件可晋升 L4：
- [ ] 负责 3 个以上大型复杂项目（>5000 节点或 2D 模型）
- [ ] 成功完成 3 个以上模型校准项目
- [ ] 发表 1 篇以上技术文章或内部技术分享
- [ ] 获得客户的正面评价
- [ ] 培养至少 1 名 L2 工程师至 L3
- [ ] 通过高级技术能力评估

## L4 - 高级工程师 (Senior Engineer)

### 总体描述
具备深厚的技术功底和丰富的项目经验，能处理最复杂的建模挑战。负责技术把关、质量审查和团队指导，参与公司技术战略制定。

### 能力要求详情

#### 1. 技术知识 (Knowledge)

| 知识点 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 全类型建模 | ★★★★★ | 精通 1D/2D/耦合模型，能处理特殊场景 |
| 前沿技术 | ★★★★☆ | 了解 AI、实时预报、数字孪生等新技术 |
| 不确定性量化 | ★★★★☆ | 掌握蒙特卡洛、GLUE 等不确定性分析方法 |
| 气候变化 | ★★★★☆ | 理解气候变化对水力建模的影响和适应方法 |
| 系统优化 | ★★★☆☆ | 了解优化算法在排水系统中的应用 |
| 软件开发 | ★★★☆☆ | 能开发建模辅助工具和插件 |
| 行业深度洞察 | ★★★★☆ | 对行业发展趋势有深入理解 |

**学习路径：**
- 研究前沿技术论文
- 开发专业工具或插件
- 参与行业标准和规范制定
- 参加高级技术论坛

#### 2. 实践技能 (Skills)

| 技能项 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 复杂问题解决 | ★★★★★ | 能解决行业级难题，提出创新方案 |
| 技术架构设计 | ★★★★☆ | 能设计复杂项目的技术架构 |
| 质量审查 | ★★★★★ | 能进行全面深入的技术审查 |
| 技术决策 | ★★★★☆ | 能做出关键的技术决策和风险评估 |
| 工具开发 | ★★★☆☆ | 开发提升效率的自动化工具 |
| 跨领域整合 | ★★★★☆ | 整合水力、水质、水生态等多领域模型 |
| 应急响应 | ★★★★☆ | 能快速响应紧急技术问题 |

**典型任务：**
- 负责公司级重点项目的技术把关
- 进行关键模型的最终质量审查
- 解决团队遇到的技术瓶颈
- 开发建模工具和方法论
- 参与行业技术交流
- 制定公司技术标准

#### 3. 项目经验 (Experience)

| 指标 | 要求 |
|------|------|
| 参与项目数 | ≥15 个 |
| 负责复杂项目数 | ≥8 个（大型或特殊项目） |
| 校准验证项目数 | ≥5 个 |
| 项目角色 | 技术负责人/审核人 |
| 项目类型 | 大型综合项目、创新项目 |
| 工作年限 | 6-10 年 |

#### 4. 质量意识 (Quality)

- ☑️ 建立公司级质量管理体系
- ☑️ 制定模型验证标准和检查清单
- ☑️ 进行最终技术审查和签字
- ☑️ 推动质量文化和技术卓越
- ☑️ 处理重大技术质量问题
- ☑️ 建立知识管理和复用机制

#### 5. 协作沟通 (Communication)

- ☑️ 与客户高层进行技术对话
- ☑️ 向公司管理层汇报技术状况
- ☑️ 培训和指导 L3 工程师
- ☑️ 在行业会议上发表演讲
- ☑️ 撰写技术白皮书或标准
- ☑️ 建立外部专家网络

### L4 晋级要求

完成以下全部条件可晋升 L5：
- [ ] 负责 5 个以上行业标杆项目
- [ ] 在行业内建立专业声誉（演讲、论文、标准）
- [ ] 培养至少 2 名 L3 工程师至 L4
- [ ] 开发创新方法或工具并被团队采用
- [ ] 获得行业认可（奖项、专家库成员等）
- [ ] 通过专家级技术能力评估

## L5 - 专家工程师 (Expert Engineer)

### 总体描述
行业内公认的权威专家，在水力建模领域具有战略影响力。负责技术战略规划、创新引领，为公司和市场提供最高级别的技术决策支持。

### 能力要求详情

#### 1. 技术知识 (Knowledge)

| 知识点 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 全领域精通 | ★★★★★ | 在水力建模全领域具有权威水平 |
| 战略技术视野 | ★★★★★ | 能预判 5-10 年技术发展趋势 |
| 跨学科整合 | ★★★★★ | 精通水力、AI、IoT、数字孪生等融合技术 |
| 科研能力 | ★★★★☆ | 能领导科研项目，发表高水平论文 |
| 标准制定 | ★★★★★ | 主导行业标准和规范的制定 |
| 国际视野 | ★★★★☆ | 了解国际最新进展，参与国际合作 |

**持续学习：**
- 跟踪顶级期刊和会议
- 参与国际合作项目
- 指导研究生或博士后
- 领导技术预研项目

#### 2. 实践技能 (Skills)

| 技能项 | 要求等级 | 具体要求 |
|--------|---------|---------|
| 战略决策 | ★★★★★ | 做出影响公司战略的技术决策 |
| 创新引领 | ★★★★★ | 引领行业技术创新方向 |
| 复杂系统 | ★★★★★ | 设计和实施超大规模复杂系统 |
| 危机处理 | ★★★★★ | 处理重大技术危机和争议 |
| 方法论 | ★★★★★ | 建立行业级方法论和最佳实践 |
| 技术传播 | ★★★★★ | 通过多种渠道传播技术思想 |

**典型任务：**
- 制定公司 3-5 年技术战略
- 处理重大技术争议和危机
- 领导前沿技术预研
- 主导行业标准制定
- 培养高级技术人才
- 参与政府政策咨询

#### 3. 项目经验 (Experience)

| 指标 | 要求 |
|------|------|
| 项目总数 | ≥25 个 |
| 标杆项目数 | ≥10 个（行业影响力） |
| 创新项目数 | ≥3 个（技术突破） |
| 项目角色 | 技术总监/首席专家 |
| 工作年限 | ≥10 年 |

#### 4. 质量意识 (Quality)

- ☑️ 定义行业质量标准
- ☑️ 处理行业级技术质量争议
- ☑️ 推动行业质量文化变革
- ☑️ 建立跨组织质量管理体系
- ☑️ 引领技术伦理和职业操守

#### 5. 协作沟通 (Communication)

- ☑️ 与行业顶级专家对话
- ☑️ 向政府提供技术政策建议
- ☑️ 在行业顶级会议发表主旨演讲
- ☑️ 建立行业专家网络和影响力
- ☑️ 指导培养 L4 工程师
- ☑️ 参与国际技术组织和标准机构

### L5 维持要求

维持 L5 等级需持续满足：
- [ ] 每年至少 1 次行业级技术分享
- [ ] 每 2 年至少 1 篇技术论文或标准
- [ ] 持续参与重大技术项目
- [ ] 保持行业影响力和声誉
- [ ] 完成规定的继续教育学时

### A.2 面试评估指南

| 等级 | 面试重点 |
|------|---------|
| L1→L2 | 基础理论、工具操作、学习能力 |
| L2→L3 | 项目经验、问题解决、验证能力 |
| L3→L4 | 复杂项目、技术创新、团队贡献 |

### A.3 实操考核标准

| 等级 | 考核内容 | 时间 | 通过标准 |
|------|---------|------|---------|
| L1→L2 | 数据整理+简单模型操作 | 4小时 | 完成基本任务 |
| L2→L3 | 中型模型构建+校准 | 8小时 | 模型通过验证 |
| L3→L4 | 复杂问题解决+方案设计 | 2天 | 方案合理可行 |
| L4→L5 | 综合项目评估 | 面试+案例 | 专家委员会认可 |
