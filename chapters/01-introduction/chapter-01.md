# 第1章 水力模型概述

## 1.1 什么是水力模型

### 1.1.1 水力模型的定义

水力模型（Hydraulic Model）是对真实排水系统水力行为的数学抽象和计算机模拟。它通过数学方程描述水流在管道、河道、泵站等设施中的运动规律，借助计算机进行求解，从而预测和分析排水系统在各种条件下的表现。

从本质上讲，水力模型是连接"真实世界"与"数字世界"的桥梁。它将复杂的物理现象转化为可计算、可分析、可优化的数字形式，为排水系统的规划、设计、运营提供科学依据。

**水力模型的核心价值在于：**
- **预测能力**：预测未来场景（如暴雨事件）下的系统表现
- **分析能力**：深入理解系统运行机制，识别瓶颈和问题
- **优化能力**：评估不同方案，找到最优解
- **沟通工具**：将复杂的技术问题可视化，便于决策沟通

### 1.1.2 水力模型的分类

水力模型可以从多个维度进行分类：

#### 按维度分类

| 维度 | 代表模型 | 适用场景 | 特点 |
|------|---------|---------|------|
| **0D** | 集总式模型 | 快速估算 | 不考虑空间分布，计算简单 |
| **1D** | SWMM, InfoWorks | 管网系统 | 沿管道方向模拟，最常用 |
| **2D** | TUFLOW, MIKE 21 | 地表淹没 | 平面模拟，显示淹没范围 |
| **3D** | FLOW-3D, OpenFOAM | 复杂结构 | 立体模拟，计算复杂 |
| **1D/2D耦合** | InfoWorks 2D, TUFLOW | 综合应用 | 管网+地表联合模拟 |

#### 按用途分类

- **规划模型**：用于排水专项规划，评估系统整体能力和问题
- **设计模型**：用于工程设计，优化管径、泵站等设施参数
- **评估模型**：用于现状评估，诊断问题，提出改造建议
- **实时预报模型**：与智慧水务结合，进行实时洪水预警和调度

#### 按方法分类

- **确定性模型**：基于物理定律，方程明确，结果可重复
- **概念性模型**：简化物理过程，注重实用性
- **数据驱动模型**：基于机器学习，从数据中学习规律

### 1.1.3 水力模型的组成要素

一个完整的水力模型通常包含以下要素：

#### 几何要素
- **管网拓扑**：节点（检查井）和管段的连接关系
- **断面形状**：圆形、矩形、马蹄形等
- **地形数据**：高程信息，影响水流方向

#### 水力要素
- **糙率系数**：反映管壁粗糙程度，常用曼宁系数n
- **边界条件**：系统边界的水位或流量条件
- **初始条件**：模拟开始时的系统状态

#### 水文要素
- **降雨输入**：设计暴雨或历史降雨序列
- **径流计算**：降雨转化为径流的方法
- **汇流过程**：径流从地表汇入管网的过程

#### 设施要素
- **泵站**：抽升能力、启停规则
- **闸门/阀门**：控制流量和水位的设施
- **调蓄池**：临时存储雨水的设施
- **溢流设施**：防止系统超载的设施

## 1.2 水力模型的发展历史

### 1.2.1 早期发展（1950s-1980s）

水力模型的发展始于20世纪50年代，经历了从物理模型到数值模型的转变。

**物理模型时代**
早期的水力研究主要依靠物理模型试验。工程师按比例缩小建造排水系统模型，通过实际观测水流运动来研究水力特性。这种方法直观可靠，但成本高、周期长、难以修改。

**数值方法的兴起**
随着计算机技术的发展，数值方法开始应用于水力计算。1950-1960年代，有限差分法等数值方法逐渐成熟，使得在计算机上求解圣维南方程成为可能。

**代表性工作**
- 斯坦福模型（Stanford Watershed Model, 1960s）：早期的流域水文模型
- 芝加哥方法：城市雨洪计算的经验方法
- SWMM的前身模型：美国环保局开始研发城市雨洪模型

### 1.2.2 软件化时代（1980s-2000s）

1980年代是个人计算机普及的时代，水力建模软件开始蓬勃发展。

**商业软件兴起**
- **SWMM**：美国环保局于1971年发布，历经多次更新，成为最流行的开源城市雨洪模型
- **MOUSE**：丹麦DHI公司开发，后演变为MIKE系列
- **InfoWorks**：英国Wallingford公司开发，功能强大的商业软件
- **HEC-RAS**：美国陆军工程兵团开发，河流分析的标准工具

**GUI革命**
1990年代，图形用户界面（GUI）的普及使水力建模从命令行时代进入可视化时代。工程师可以通过点击鼠标构建模型，大大降低了使用门槛。

**标准化进程**
这一时期，各国开始制定水力建模相关的标准和规范：
- 美国：FEMA发布洪水建模指南
- 英国：WaPUG（后并入CIWEM）发布建模规范
- 澳大利亚：发布Australian Rainfall and Runoff

### 1.2.3 集成化时代（2000s-2020s）

21世纪的水力模型朝着集成化、多维化发展。

**多维耦合**
- **1D/2D耦合模型**：同时模拟管网流动和地表淹没，如TUFLOW、InfoWorks ICM
- **水力-水质耦合**：同时模拟水流和污染物传输
- **水力-水生态耦合**：扩展至生态系统影响评估

**实时控制集成**
实时控制（RTC）系统与水力模型的集成，使得模型可以指导实际设施的运营调度：
- 泵站智能调度
- 闸门实时控制
- 预警预报系统

**数字孪生概念**
数字孪生（Digital Twin）概念的引入，将水力模型从"离线分析工具"升级为"实时映射系统"：
- 实时数据接入（IoT传感器）
- 模型实时更新
- 预测性维护

### 1.2.4 智能化时代（2020s-今）

当前，人工智能技术正在深刻改变水力建模领域。

**机器学习应用**
- **参数识别**：用机器学习从数据中提取模型参数
- **结果预测**：用神经网络快速预测水力响应
- **异常检测**：自动识别监测数据中的异常

**大语言模型**
- **智能助手**：辅助建模、解答问题、生成报告
- **知识问答**：基于专业文档的智能问答系统
- **代码生成**：自动生成数据处理脚本

**AI智能体**
- **自主建模**：AI Agent自动完成建模流程
- **人机协作**：AI作为"第六个团队成员"与人类协作

## 1.3 水力模型的核心机理

### 1.3.1 水力学基础原理

水力模型的核心是描述水流运动的物理定律，最主要的是圣维南方程组（Saint-Venant Equations）。

#### 连续性方程

连续性方程表达质量守恒原理：

$$
\frac{\partial A}{\partial t} + \frac{\partial Q}{\partial x} = q
$$

其中：
- $A$：过水断面面积（m²）
- $Q$：流量（m³/s）
- $t$：时间（s）
- $x$：沿流程距离（m）
- $q$：侧向入流（m²/s）

**物理意义**：控制体内水量的变化率等于流入与流出的流量差。

#### 动量方程

动量方程表达牛顿第二定律在水流中的应用：

$$
\frac{\partial Q}{\partial t} + \frac{\partial (Q^2/A)}{\partial x} + gA\frac{\partial h}{\partial x} + gA S_f = 0
$$

其中：
- $g$：重力加速度（9.81 m/s²）
- $h$：水深（m）
- $S_f$：摩擦坡度

**物理意义**：水流动量的变化率等于压力、重力和摩擦力之和。

#### 曼宁公式

曼宁公式用于计算摩擦坡度，是最常用的阻力公式：

$$
Q = \frac{1}{n} A R^{2/3} S^{1/2}
$$

其中：
- $n$：曼宁粗糙系数
- $R$：水力半径（m），$R = A/P$，$P$为湿周
- $S$：能坡或底坡

**曼宁系数n的典型值**：

| 管道材质 | n值范围 |
|---------|--------|
| 混凝土管 | 0.013-0.015 |
| 钢筋混凝土管 | 0.013-0.015 |
| 铸铁管 | 0.013-0.015 |
| PVC管 | 0.009-0.011 |
| 砖砌渠道 | 0.015-0.017 |
| 土渠 | 0.020-0.030 |

### 1.3.2 水文学基础原理

水力模型需要水文输入，核心是降雨-径流转化。

#### 径流系数法

最简单的径流计算方法：

$$
Q = \psi \cdot q \cdot A
$$

其中：
- $Q$：设计流量（L/s）
- $\psi$：径流系数
- $q$：设计暴雨强度（L/(s·hm²)）
- $A$：汇水面积（hm²）

**径流系数参考值**：

| 地表类型 | 径流系数 |
|---------|---------|
| 屋顶、混凝土 | 0.85-0.95 |
| 沥青路面 | 0.85-0.90 |
| 块石路面 | 0.70-0.85 |
| 草地 | 0.10-0.25 |
| 林地 | 0.05-0.15 |

#### 设计暴雨

设计暴雨是水力模型的重要输入，通常用强度-历时-频率（IDF）曲线表示：

$$
q = \frac{a}{(t+b)^c}
$$

其中：
- $q$：降雨强度（mm/h）
- $t$：降雨历时（min）
- $a, b, c$：与地区和重现期相关的参数

**Python代码示例：计算设计暴雨**

```python
def calculate_design_storm(duration, return_period, params):
    """
    计算设计暴雨强度
    
    参数:
        duration: 降雨历时(分钟)
        return_period: 重现期(年)
        params: 包含a, b, c参数的字典
    
    返回:
        降雨强度(mm/h)
    """
    a = params['a']
    b = params['b'] 
    c = params['c']
    
    intensity = a / ((duration + b) ** c)
    return intensity

# 示例：北京市IDF参数(2年重现期)
beijing_params = {
    'a': 11.02,
    'b': 8.5,
    'c': 0.71
}

# 计算历时60分钟的设计暴雨
q = calculate_design_storm(60, 2, beijing_params)
print(f"设计暴雨强度: {q:.2f} mm/h")
```

### 1.3.3 数值求解方法

圣维南方程是偏微分方程组，通常需要用数值方法求解。

#### 有限差分法

将时间和空间离散化，用差分近似微分：

$$
\frac{\partial Q}{\partial x} \approx \frac{Q_{i+1} - Q_i}{\Delta x}
$$

**特点**：
- 实现简单
- 计算效率高
- 稳定性需要满足CFL条件

#### 有限体积法

将计算域划分为控制体，在控制体上积分守恒方程：

$$
\frac{\partial}{\partial t}\int_V U dV + \oint_S F \cdot n dS = 0
$$

**特点**：
- 严格保持守恒性
- 适用于非结构化网格
- 精度较高

#### 求解流程

```
1. 初始化：设置初始条件和边界条件
    ↓
2. 时间步进：从t到t+Δt
    ↓
3. 空间离散：对每个计算单元求解
    ↓
4. 方程求解：解线性或非线性方程组
    ↓
5. 收敛判断：检查是否满足收敛条件
    ↓
6. 输出结果：保存当前时刻的结果
    ↓
7. 循环：重复步骤2-6直到结束
```

### 1.3.4 模型验证与校准原理

模型验证和校准是确保模型可靠性的关键步骤。

#### 验证与校准的区别

| 项目 | 校准（Calibration） | 验证（Verification） |
|------|-------------------|-------------------|
| 目的 | 调整参数使模型匹配观测 | 检验模型的预测能力 |
| 数据 | 使用校准数据集 | 使用独立的验证数据集 |
| 活动 | 调整参数 | 固定参数，运行模型 |

#### 统计评价指标

**纳什效率系数（NSE）**：

$$
NSE = 1 - \frac{\sum_{i=1}^{n}(O_i - S_i)^2}{\sum_{i=1}^{n}(O_i - \bar{O})^2}
$$

- NSE = 1：完美拟合
- NSE > 0.75：优秀
- 0.5 < NSE ≤ 0.75：良好
- NSE ≤ 0.5：较差

**均方根误差（RMSE）**：

$$
RMSE = \sqrt{\frac{\sum_{i=1}^{n}(O_i - S_i)^2}{n}}
$$

**Python代码示例：计算NSE和RMSE**

```python
import numpy as np

def calculate_nse(observed, simulated):
    """计算纳什效率系数"""
    observed = np.array(observed)
    simulated = np.array(simulated)
    
    numerator = np.sum((observed - simulated) ** 2)
    denominator = np.sum((observed - np.mean(observed)) ** 2)
    
    nse = 1 - (numerator / denominator)
    return nse

def calculate_rmse(observed, simulated):
    """计算均方根误差"""
    observed = np.array(observed)
    simulated = np.array(simulated)
    
    rmse = np.sqrt(np.mean((observed - simulated) ** 2))
    return rmse

# 示例数据
observed_flow = [10, 15, 25, 30, 20, 12]
simulated_flow = [9, 14, 27, 28, 22, 11]

nse = calculate_nse(observed_flow, simulated_flow)
rmse = calculate_rmse(observed_flow, simulated_flow)

print(f"NSE: {nse:.3f}")
print(f"RMSE: {rmse:.3f}")
```

## 1.4 水力模型的作用与意义

### 1.4.1 在规划阶段的作用

水力模型是排水专项规划的核心工具。

**现状评估与问题诊断**
- 识别系统瓶颈和薄弱环节
- 量化内涝风险和溢流污染
- 评估系统应对极端事件的能力

**规划方案比选与优化**
- 评估不同规划方案的效果
- 优化管网布局和设施规模
- 确定分期建设时序

**案例：某新区排水专项规划**
某新区规划面积100平方公里，规划人口80万。通过水力模型：
- 识别出12处排水瓶颈
- 评估了3个规划方案
- 优化确定了管网规模和泵站配置
- 节省工程投资约15%

### 1.4.2 在设计阶段的作用

水力模型优化工程设计，提高工程质量和经济性。

**管径优化与平面布置**
- 确定经济合理的管径
- 优化管道坡度和埋深
- 减少施工难度和成本

**泵站规模与调度策略**
- 确定泵站设计规模
- 优化水泵配置
- 制定调度控制策略

**调蓄设施设计**
- 确定调蓄池容积
- 优化进出水管设计
- 评估放空时间

**案例：某易涝点整治工程设计**
某易涝点历史上每年发生3-4次积水。通过水力模型：
- 诊断出上游管径不足和下游顶托两个问题
- 设计了管网改造+局部调蓄的综合方案
- 模拟验证了方案效果：可抵御50年一遇暴雨

### 1.4.3 在运营阶段的作用

水力模型与智慧水务结合，支撑排水系统智慧运维。

**实时洪水预警**
- 接入实时降雨和监测数据
- 滚动预测未来2-6小时的积水风险
- 提前发布预警信息

**泵站智能调度**
- 根据实时工况优化泵站运行
- 平衡能耗和排水效果
- 减少溢流污染

**管网堵塞预警**
- 分析流量监测数据
- 识别异常流量模式
- 提前发现堵塞隐患

**案例：某市智慧水务数字孪生应用**
某市建立排水系统数字孪生平台：
- 实时接入2000+监测点数据
- 每5分钟滚动更新模型计算
- 预警准确率85%以上
- 内涝响应时间从2小时缩短到30分钟

### 1.4.4 经济效益与社会价值

**直接经济效益**
- 减少工程投资：通过优化设计，通常可节省10-20%投资
- 降低运营成本：智能调度可节省20-30%电耗
- 减少灾害损失：提前预警可减少内涝损失50%以上

**社会价值**
- 提升城市安全：减少内涝灾害，保障人民生命财产安全
- 改善环境质量：减少溢流污染，改善水环境
- 提高管理水平：实现科学决策和精细化管理

**ROI分析方法**

```
ROI = (收益 - 成本) / 成本 × 100%

其中：
- 成本：建模费用、软件费用、人员费用
- 收益：节省投资 + 降低运营成本 + 减少灾害损失

典型案例ROI：300-500%（3-5年回报）
```
